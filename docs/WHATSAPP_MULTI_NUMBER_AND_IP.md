# Несколько номеров WhatsApp: один IP или разные внешние адреса?

Кратко для ChatSwitch (Laravel + **whatsapp-service** на Node с **whatsapp-web.js**).

## Можно ли подключить несколько номеров с одного IP?

**Да, технически это возможно и типично.** На одном сервере (один публичный IPv4/IPv6) вы поднимаете `whatsapp-service`, а внутри него — **отдельный клиент на каждую сессию** (`session_name` + отдельная папка авторизации). Все эти клиенты выходят в интернет с **одного и того же IP** вашего VPS/хостинга.

Так делают тысячи офисов: много телефонов за одним NAT.

## Нужны ли отдельные внешние IP на каждый номер?

**Нет, не требуется** для того, чтобы «просто подключить несколько номеров» в рамках одного инстанса сервиса.

Отдельные IP / прокси имеют смысл только как **опциональная** мера, если:

- вы массово автоматизируете рассылки и хотите **распределить риск блокировок**;
- провайдер или политика безопасности требуют **изоляции** трафика;
- вы масштабируете **много инстансов** `whatsapp-service` на разных машинах (у каждой свой IP).

Это не часть официального API WhatsApp и не гарантирует защиту от ограничений со стороны Meta.

## Официальная позиция WhatsApp / Meta

**WhatsApp Business Platform (Cloud API)** — рекомендуемый корпоративный путь с договором и лимитами.  
**whatsapp-web.js** — неофициальный клиент веб-версии; при большом объёме или нарушении правил возможны **ограничения и блокировки**, независимо от количества IP.

## Практические рекомендации для одного IP

1. **Не создавать лишние сессии** — только реальные рабочие номера.  
2. **Не хранить в БД «фейковые» подключённые сессии** без живого клиента в `whatsapp-service` (в ChatSwitch статусы на странице подключений синхронизируются с микросервисом).  
3. При проблемах смотреть логи **whatsapp-service** и не смешивать демо-данные с продакшеном.

## Итог

| Вопрос | Ответ |
|--------|--------|
| Несколько номеров с одного IP? | Да, обычная схема для одного VPS + одного `whatsapp-service`. |
| Обязательны разные внешние IP? | Нет. |
| Когда думают про прокси / разные IP? | Высокая нагрузка, риск блокировок, разнесение инстансов по серверам — по согласованию с политикой и правилами Meta. |
