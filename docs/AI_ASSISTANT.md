# AI-ассистент оператора

Встроенный AI-помощник в окне диалога. Анализирует переписку клиент ↔ оператор и подсказывает готовые формулировки ответов в стиле операторов, уже работающих с этим клиентом.

---

## Как выглядит

Правая выдвижная панель в окне чата (`AiAssistantPanel.vue`). Открывается кнопкой в шапке диалога. Поверх основной переписки с клиентом — отдельный чат оператора с AI, недоступный клиенту.

---

## Архитектура

```
Оператор → AiAssistantPanel.vue
    → POST /chats/{chat}/ai/chat   (throttle: 30 req/min)
    → ChatAiAssistantController
    → ChatAssistantService
        → OpenAiChatService         → OpenAI Chat Completions API
```

### `ChatAssistantService`

Файл: `app/Services/AI/ChatAssistantService.php`

| Параметр | Значение |
|----------|---------|
| Контекст истории переписки | до 80 последних сообщений (`HISTORY_LIMIT`) |
| Обрезка тела сообщения | 600 символов (`HISTORY_BODY_TRUNCATE`) |
| История диалога оператора с AI | до 20 ходов (`ASSISTANT_HISTORY_LIMIT`) |

Формирует два системных промпта:

1. **Системный промпт** — объясняет AI его роль, задачи, ограничения. Указывает имя оператора и контакта клиента.
2. **Контекст переписки** — хронологическая выгрузка чата с разметкой «Оператор / Клиент».

Подпись оператора (вида `*Имя (Роль)*\n`) из исходящих сообщений автоматически вырезается через `OperatorSignature::strip()`, чтобы AI не копировал её в подсказках.

### `OpenAiChatService`

Файл: `app/Services/AI/OpenAiChatService.php`

Тонкий HTTP-клиент над OpenAI Chat Completions. Конфигурация через `.env`:

| Переменная | По умолчанию | Описание |
|-----------|--------------|----------|
| `OPENAI_API_KEY` | — | Обязательна |
| `OPENAI_BASE_URL` | `https://api.openai.com/v1` | Можно переключить на прокси |
| `OPENAI_MODEL` | `gpt-4o-mini` | Модель |
| `OPENAI_TIMEOUT` | `45` сек | Таймаут запроса |

Параметры запроса в `ChatAssistantService::reply`: `temperature=0.45`, `max_tokens=900`.

### `ChatAiAssistantController`

Файл: `app/Http/Controllers/ChatAiAssistantController.php`

- Проверяет право `view` на чат через Policy.
- Принимает: `message` (до 4000 симв.) + `history` (массив до 40 ходов).
- При ошибке OpenAI возвращает `502` с описанием, при нераспознанном исключении — `500`.

## Хранение истории на фронте

История диалога оператора с AI живёт в `localStorage` под ключом:

```
chatswitch:ai-assistant:{chatId}
```

Хранится до 40 последних ходов. При повторном открытии панели — история восстанавливается. Очищается вручную кнопкой «Очистить» в панели. При переключении на другой чат — история своя (ключ привязан к `chatId`).

## Быстрые действия (Quick Actions)

Три заготовленных промпта для частых сценариев:

| Кнопка | Промпт |
|--------|--------|
| Подскажи ответ клиенту | Анализ последней реплики клиента + 1–2 варианта ответа в стиле операторов |
| О чём этот диалог? | Краткий пересказ: что хочет клиент, что уже ответили, открытые вопросы |
| Возражения клиента | Список возражений и сомнений + способы их закрыть в стиле компании |

## Настройка

1. Получите API-ключ OpenAI.
2. Добавьте в `.env`:

```env
OPENAI_API_KEY=sk-...
OPENAI_MODEL=gpt-4o-mini
# опционально — если используете совместимый прокси:
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_TIMEOUT=45
```

3. Убедитесь, что конфиг `config/services.php` содержит секцию `openai`:

```php
'openai' => [
    'api_key'  => env('OPENAI_API_KEY'),
    'base_url' => env('OPENAI_BASE_URL', 'https://api.openai.com/v1'),
    'model'    => env('OPENAI_MODEL', 'gpt-4o-mini'),
    'timeout'  => env('OPENAI_TIMEOUT', 45),
],
```

---

## Права доступа

AI-ассистент доступен любому аутентифицированному пользователю, у которого есть право `view` на чат (определяется через `ChatPolicy`). Клиент **никогда** не видит переписку оператора с AI.
