# План улучшений AI, UI и UX

## Цель

Сделать AI-ассистента понятным, управляемым и безопасным для операторов и администраторов: пользователь должен понимать, когда AI отвечает, почему он ответил именно так, какие данные использовал и что нужно настроить для качественных ответов.

## Принципы

- Не показывать обычным пользователям SQL, stack trace и технические детали.
- Любое действие AI должно быть объяснимым: ответил, не ответил, подготовил черновик, ждёт данных.
- Админка должна подсказывать, что настроено плохо или не настроено вообще.
- Оператор должен быстро исправлять AI-ответ без перехода между разделами.
- Критичные действия должны быть обратимыми или требовать явного подтверждения.

## Приоритет 1: Доверие к AI

### 1. AI-статусы в чате

Статус: частично выполнено 2026-05-13.

Сделано:

- В шапке чата режим отображается как `AI авто` / `AI выключен`.
- Кнопка AI показывает понятные состояния: `AI`, `AI думает`, `AI ошибка`, `AI стоп`.
- Для ошибки и процесса генерации добавлена отдельная визуальная окраска кнопки.

Осталось:

- Добавить полноценный раскрываемый блок статуса с историей последних AI-решений.
- Добавить режим `AI черновик`, когда AI готовит текст без автоотправки.

Добавить понятные состояния в шапке или панели чата:

- `AI авто` — AI сам отвечает на новые сообщения.
- `AI черновик` — AI готовит ответ, оператор отправляет вручную.
- `AI выключен` — AI не участвует в диалоге.
- `AI ошибка` — AI недоступен, но без технического текста для пользователя.

Для каждого статуса добавить короткую подсказку: что произойдёт при новом сообщении клиента.

### 2. Причина ответа или отказа

Статус: частично выполнено 2026-05-13.

Сделано:

- `ChatController` отдаёт для последнего AI-лога поля `label`, `message`, `hint`.
- `ChatHeader.vue` показывает безопасное описание статуса в tooltip AI-кнопки.
- `AiAssistantPanel.vue` показывает видимый блок "Последнее решение AI" с результатом, подсказкой и временем.
- В AI-панели показываются текущие счётчики базы знаний: правила, товары и услуги, доступные для промпта.
- Техническая ошибка отдаётся как `technical_error` только администратору.

Осталось:

- Добавить источник тона: сотрудник / компания / не собран.

В панели AI показывать компактный блок:

- `Ответ подготовлен по базе знаний`.
- `Использовано: 2 правила, 1 товар, 0 услуг`.
- `Профиль тона: сотрудник / компания / не собран`.
- `AI не ответил: нет компании у чата`.
- `AI не ответил: нет активных правил в базе знаний`.

Технические детали показывать только администратору в блоке `Подробнее`.

### 3. Безопасные ошибки AI

Статус: выполнено для последнего AI-статуса в чате 2026-05-13.

Сделано:

- SQL/табличные ошибки заменяются на безопасный текст для интерфейса.
- Ошибки AI/API/timeout заменяются на человекочитаемое сообщение.
- Полная техническая причина остаётся доступна администратору через `technical_error`.
- Ручной endpoint AI-панели (`chats.ai.chat`) больше не возвращает сырой текст исключения обычному пользователю.
- AI-панель не использует низкоуровневый `e.message` как fallback для toast/черновика.

Осталось:

- Проверить остальные публичные AI/toast сценарии на наличие сырых технических ошибок.

Заменить сырые ошибки вида `SQLSTATE...` на человекочитаемые сообщения:

- `AI временно недоступен. Администратору нужно проверить настройки базы знаний.`
- `AI не смог подготовить ответ. Попробуйте ещё раз или ответьте вручную.`

В логах при этом сохранять полную техническую причину.

## Приоритет 2: Скорость оператора

### 1. Режим черновика

Добавить режим, где AI не отправляет ответ сразу, а вставляет его в поле ввода как черновик.

Особенно важно для:

- цен;
- записи на услугу;
- претензий;
- конфликтных сообщений;
- вопросов, где база знаний неполная.

### 2. Кнопка ручного AI-ответа

Рядом с полем ввода добавить действие:

- `AI ответить`;
- `AI улучшить текст`;
- `AI сделать короче`;
- `AI сделать вежливее`.

Ответ должен попадать в поле ввода, чтобы оператор мог проверить и отправить.

### 3. Визуальное отличие AI-сообщений

AI-сообщения в истории чата должны иметь небольшой индикатор:

- значок `AI`;
- tooltip `Ответ подготовлен AI`;
- ссылка на запись лога или блок причины, если пользователь администратор.

## Приоритет 3: Админка базы знаний

### 1. Чеклист готовности AI

На странице базы знаний добавить блок готовности по компании:

- правила ответа есть;
- есть товары, включённые в промпт;
- есть услуги, включённые в промпт;
- общий тон компании собран;
- есть хотя бы один сотрудник с профилем тона;
- тестовый вопрос проверен.

Статусы: `готово`, `нужно настроить`, `необязательно`.

### 2. Предупреждения по данным

Показывать предупреждения:

- `Нет активных правил ответа`.
- `Все товары выключены из промпта`.
- `Все услуги выключены из промпта`.
- `У товара нет цены`.
- `У услуги нет длительности`.
- `Есть правила с одинаковым приоритетом`.

### 3. Тестовый вопрос

Добавить форму:

- выбрать компанию;
- ввести вопрос клиента;
- получить тестовый AI-ответ;
- показать использованные правила, товары, услуги и профиль тона.

Тестовый ответ не должен отправляться в чат.

### 4. Аудит изменений

Для товаров, услуг и правил ответа хранить:

- кто создал;
- кто изменил;
- когда изменил;
- какие поля изменились.

Минимальный вариант: `updated_by_user_id` и история последних изменений.

## Приоритет 4: Общий UI/UX

### 1. Единые пустые состояния

На всех пустых страницах использовать шаблон:

- короткое объяснение;
- одна главная кнопка действия;
- 1–2 ссылки на связанные разделы.

Примеры:

- нет товаров → `Добавить товар`;
- нет правил → `Добавить правило ответа`;
- нет клиентов → `Добавить клиента`;
- нет подключений WhatsApp → `Подключить номер`.

### 2. Поиск по настройкам

Расширить поиск в левом меню с синонимами:

- `прайс`, `цена`, `каталог` → `Товары`;
- `запись`, `длительность`, `услуга` → `Услуги`;
- `AI`, `правила`, `тон`, `ответ` → `База знаний`;
- `номер`, `QR`, `WhatsApp` → `Подключения WhatsApp`.

### 3. Подтверждение удаления

Для важных сущностей просить ввести название:

- товар;
- услуга;
- правило ответа;
- компания;
- пользователь;
- подключение WhatsApp.

Для менее важных действий достаточно обычного confirm.

### 4. Единые уведомления

Уведомления должны отвечать на вопрос пользователя:

- что произошло;
- что делать дальше, если ошибка;
- нужно ли обновлять страницу.

Пример:

`Запись сохранена. AI начнёт учитывать её сразу, если включено «В промпте».`

## Приоритет 5: Метрики качества

### 1. Оценка AI-ответа

После AI-ответа добавить простую оценку:

- `Хорошо`;
- `Не тот стиль`;
- `Факт неверный`;
- `Слишком длинно`;
- `Не хватило базы знаний`.

Эти события сохранять в лог AI-ответов.

### 2. Панель проблем AI

Для админа добавить список:

- последние ошибки AI;
- чаты, где AI не ответил;
- частые причины отказов;
- ответы с плохой оценкой;
- компании без базы знаний.

## Порядок внедрения

### Этап 1: Быстрые улучшения

- Сделать безопасное отображение AI-ошибок.
- Добавить понятные AI-статусы в шапку чата.
- Показывать, почему AI ответил или не ответил.
- Завершить предпросмотр AI-контекста в базе знаний.

### Этап 2: Управление ответом

- Добавить режим черновика.
- Добавить кнопку ручного AI-ответа.
- Визуально помечать AI-сообщения.
- Добавить оценку AI-ответа.

### Этап 3: Админская диагностика

- Добавить чеклист готовности AI.
- Добавить тестовый вопрос.
- Добавить предупреждения по неполной базе знаний.
- Добавить аудит изменений базы знаний.

### Этап 4: Полировка UX

- Унифицировать пустые состояния.
- Расширить поиск по настройкам.
- Усилить подтверждения удаления.
- Привести уведомления к единому формату.

## Критерии готовности

- Оператор понимает текущий режим AI без открытия настроек.
- Администратор видит, почему AI отвечает плохо или не отвечает.
- Технические ошибки не показываются обычному пользователю.
- AI-ответ можно подготовить как черновик и вручную отправить.
- В базе знаний видно, какие данные реально попадут в промпт.
- Все критичные UX-сценарии покрыты smoke-тестами или feature-тестами.
