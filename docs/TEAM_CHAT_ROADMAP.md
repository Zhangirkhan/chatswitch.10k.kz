## 1. Текущее состояние (кратко)

- Отдельный домен **team chat**: таблицы `team_conversations`, `team_conversation_user`, `team_messages` (не смешивается с WhatsApp `chats` / `messages`).
- UI во вкладке **Организация**: подразделы «Задачи» / «Чат», беседы, контакты компании, ЛС и чаты отделов.
- Состав **чатов отделов** синхронизируется с `department_user`; выход из группы отдела не предусмотрен; состав меняется через управление отделами/пользователями.
- Realtime: Laravel Echo, каналы `team-conversation.{id}`, `team-inbox.{userId}`; **`team.read`**, **`team.delivered`** (ЛС и отделы).
- **Пересылка:** поля `forwarded_from_team_message_id`, денормализованный превью в `team_messages`; API и UI.
- **Поиск:** `GET /organization/chat/api/search?q=` — беседы, сообщения (`body`, цитата пересылки) и **коллеги** компании по имени/email (`colleagues`); в сайдбаре поле поиска; переход к сообщению с `?highlight_message_id=`.
- **Доставлено / прочитано (ЛС):** в pivot `team_conversation_user.last_delivered_message_id`; `POST .../delivered`; Echo `team.delivered`; подписи «Отправлено» / «Доставлено» / «Прочитано».
- **Доставлено в чате отдела:** тот же pivot; в `read_meta` поле **`others_min_last_delivered_message_id`** (минимум по остальным участникам); в UI для своих сообщений «Доставлено», когда все остальные отметили доставку до этого id.
- **Упоминания:** JSON + **`team_message_mentions`**; кнопки «Упомянуть»; подсветка `@Имя`; при разрешении **Notification API** — системное уведомление только упомянутым, если вкладка в фоне; серверный e-mail / mobile push — ⬜.
- **Закреп:** в pivot `team_conversation_user.pinned_at` (лично на пользователя); `POST .../api/{conversation}/pin` с `{ pinned: bool }`; в списке бесед закреплённые сверху, сортировка по `pinned_at` среди закреплённых.
- **Закреп сообщения в чате отдела (общее для всех):** поле `team_conversations.pinned_team_message_id` → `team_messages`; только тип `department`; менять могут **администратор и менеджер** (`pinRoomMessage`); `POST .../api/{conversation}/pinned-message` с `{ team_message_id: int|null }` (`present` + `nullable`); в ответе `messages` — `conversation.room_pinned_message` и `can_pin_room_message`; Echo `team.room-pin`; при soft-delete сообщения закреп снимается.
- **Ответы (MVP, один уровень):** в `team_messages` поле `parent_team_message_id` (только на **корневое** сообщение в беседе; ответ на ответ API отклоняет); лента плоская, в UI блок цитаты по полю `reply_to`; Echo `TeamMessageReceived` передаёт `parent_team_message_id` и `reply_to`; пересылка вместе с `parent_team_message_id` запрещена.
- Доступ к разделу организации (включая чат) завязан на модуль **`module_tasks`** (как и остальная «Организация»).

## 3. Дополнительные идеи (расширение)

- **Черновики и отложенная отправка** — не терять текст при смене беседы.
- **Политика хранения** переписки (ретенция) + юридическое согласие.
- **Боты / webhooks** (уведомления в чат отдела из CI, CRM, мониторинга).
- **Метрики** (без чтения текста): активность по отделам, время ответа SLA.
- **Доступность**: клавиатурная навигация, screen reader.
- **PWA / офлайн-кэш** последних сообщений (осторожно с чувствительными данными).
- **RTL / мультиязычность** интерфейса.

## 4. Бэклог по категориям

### 4.1. Продукт и UX

- Черновики и отложенная отправка — не потерять текст при переключении бесед. ✅ черновики в `sessionStorage` (см. фаза 1)
- Пересылка сообщения в другой внутренний чат (с пометкой «переслано»). ✅
- Треды / ответы в ветку в группах отдела, чтобы не засорять основной поток. 🟡 один уровень (`parent_team_message_id` + цитата в UI); отдельный поток ветки, счётчики по треду, ответ на ответ — ⬜
- @упоминания с подсветкой и (опционально) push только упомянутым. 🟡 JSON + таблица + UI + браузерные уведомления при @ и фоновой вкладке (после разрешения); e-mail / mobile push — ⬜
- Статусы сообщения: доставлено / прочитано (хотя бы для ЛС). 🟡 ЛС: прочитано + `team.read`, доставлено + `team.delivered`; отделы: доставлено «всем остальным» по `others_min_last_delivered_message_id`; прочитано по группе — ⬜
- Голосовые / быстрые ответы-стикеры — если операторам привычнее голос, чем набор в WA-чате. ⬜
- Комнаты по проекту (не только отдел) — кросс-функциональные команды. ⬜

### 4.2. Поиск и навигация

- Глобальный поиск по людям, чатам и фрагментам текста из одного поля. 🟡 чаты + сообщения + коллеги компании (`colleagues`); контакты по-прежнему отдельная вкладка «Сотрудники»
- Фильтры списка бесед: только непрочитанные, только отделы, только ЛС. ✅
- Закреплённые чаты вверху списка (личная сортировка, не для всех). ✅ `pinned_at` в pivot + UI «☆/★» в сайдбаре
- Закреплённое сообщение в чате отдела (одно на беседу, видно всем участникам). ✅ колонка `team_conversations.pinned_team_message_id`, менеджер/админ, баннер над лентой, кнопка 📌 у сообщений, `team.room-pin`

### 4.3. Безопасность и комплаенс

- DLP / запрет внешних ссылок или сканирование вложений на малварь.
- Экспорт переписки для HR / расследований (с журналом: кто запросил, когда, объём).
- Watermark на вложениях (user id / время) для чувствительных компаний.

### 4.4. Надёжность и производительность

- Очередь на тяжёлые вложения + прогресс-бар; лимиты по размеру / типу на уровне политики компании.
- Идемпотентность отправки (`client_message_id`) против дублей при плохой сети. ✅
- Пагинация и индексы под рост истории; отдельное хранилище для больших blob (S3 и т.п.). 🟡 пагинация сообщений; индексы под FTS при росте — по необходимости

### 4.5. Интеграции

- Календарь: кнопка «создать встречу» из чата с предзаполненными участниками.
- Задачи организации: «создать задачу из сообщения» со ссылкой на пост.

### 4.6. Клиенты и поля

- Mobile-first layout для полевых сотрудников. 🟡 базовые правки (см. фаза 1)

## 5. Поэтапная реализация

Рекомендуемый порядок внедрения, чтобы не расползся scope и не блокировать друг друга.

### Фаза 1 — быстрый выигрыш без тяжёлой схемы БД

| Приоритет | Что делаем | Заметки | Статус |
|-------------|------------|---------|--------|
| Высокий | **Черновики** при переключении бесед | Сначала `localStorage` / `sessionStorage` по ключу `draft:team_conv:{id}`; позже — серверные черновики. | ✅ |
| Высокий | **`client_message_id`** | Поле в `team_messages` + уникальный индекс `(team_conversation_id, client_message_id)` где `client_message_id IS NOT NULL`; клиент — UUID на отправку. | ✅ |
| Средний | **Фильтры списка бесед** | UI + query-параметры к существующему API списка. | ✅ |
| Средний | **Mobile-first** | Ширины, touch targets, нижняя панель ввода; без новых сущностей. | 🟡 шире пузыри на узком экране, `touch-manipulation`, кнопки переслать/«Упомянуть» ≥40px по высоте |

### Фаза 2 — модель «как в мессенджере»

| Приоритет | Что делаем | Заметки | Статус |
|-------------|------------|---------|--------|
| Высокий | **Доставлено / прочитано** (минимум ЛС) | `last_delivered_message_id`; Echo `team.delivered`; UI для ЛС и **групп отдела** (`others_min_last_delivered_message_id`). | 🟡 ЛС полный цикл; отделы — доставлено «всем остальным» без прочитано по группе |
| Высокий | **@упоминания** | JSON + `team_message_mentions`; UI; браузерные уведомления при @ в фоне вкладки. | 🟡 без серверного e-mail/push |
| Средний | **Пересылка** | `forwarded_from_message_id` + денормализованный превью; политика «куда можно переслать». | ✅ `forwarded_from_team_message_id` + превью; одна компания + участник в источнике и цели |
| Средний | **Глобальный поиск** | Беседы, текст сообщений `LIKE`, коллеги компании в `colleagues`; при росте — внешний поиск. | 🟡 `GET .../api/search`; без Meilisearch |

### Фаза 3 — тяжёлые фичи

| Приоритет | Что делаем | Заметки | Статус |
|-------------|------------|---------|--------|
| Средний | **Треды** | `parent_team_message_id` + `reply_to` в API/Echo; один уровень (родитель только корень). Дальше: сущность `team_thread`, вложенные ответы, счётчики/уведомления по ветке. | 🟡 MVP без отдельной сущности ветки |
| Средний | **Вложения + превью ссылок** | Как в организации + очередь; затем watermark / DLP как надстройка. | ⬜ |
| Низкий | **Голосовые / стикеры** | Тип контента, хранилище, транскодирование — после стабильных вложений. | ⬜ |

### Фаза 4 — комплаенс и интеграции

| Приоритет | Что делаем | Заметки | Статус |
|-------------|------------|---------|--------|
| Средний | **Экспорт для HR** | Только роль + audit log (`who`, `when`, `scope`). | ⬜ |
| Средний | **Календарь из чата** | Deep link / draft события с `participant_ids`. | ⬜ |
| Средний | **Задача из сообщения** | POST в `organization.posts.store` с цитатой / ссылкой на сообщение. | ⬜ |

### Закрепление чатов (можно в любой фазе после списка)

- Таблица `user_pinned_team_conversations` или флаги в pivot пользователя ↔ беседа; сортировка «закреплённые сверху» только для текущего пользователя. ✅ реализовано через `team_conversation_user.pinned_at` (без отдельной таблицы)

## 7. Риски и нюансы (напоминание)

- **Пользователь без `company_id`**: ЛС и контакт-лист по компании недоступны; поведение зафиксировать в продукте.
- **Смена `company_id` у пользователя**: обновить или очистить участие в чатах старой компании.
- **Модуль организации**: сейчас чат завязан на `module_tasks`; при необходимости выделить отдельный флаг `module_team_chat`.
- **Офлайн / PWA**: баланс удобства и утечки данных на устройстве.
