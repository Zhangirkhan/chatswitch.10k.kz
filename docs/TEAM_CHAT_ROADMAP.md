## 1. Текущее состояние (кратко)

- Отдельный домен **team chat**: таблицы `team_conversations`, `team_conversation_user`, `team_messages` (не смешивается с WhatsApp `chats` / `messages`).
- UI во вкладке **Организация**: подразделы «Задачи» / «Чат», беседы, контакты компании, ЛС и чаты отделов.
- Состав **чатов отделов** синхронизируется с `department_user`; выход из группы отдела не предусмотрен; состав меняется через управление отделами/пользователями.
- Realtime: Laravel Echo, каналы `team-conversation.{id}`, `team-inbox.{userId}`; **`team.read`**, **`team.delivered`**, **`team.typing`**, **`team.message.reactions`**, **`team.room-pin`**.
- **Пересылка:** поля `forwarded_from_team_message_id`, денормализованный превью в `team_messages`; API и UI.
- **Поиск:** `GET /organization/chat/api/search?q=` — беседы, сообщения (`body`, цитата пересылки) и **коллеги** компании по имени/email (`colleagues`); в сайдбаре поле поиска; переход к сообщению с `?highlight_message_id=`.
- **Доставлено / прочитано (ЛС):** в pivot `team_conversation_user.last_delivered_message_id` / `last_read_message_id`; `POST .../delivered`, `POST .../read`; Echo `team.delivered` / `team.read`; подписи «Отправлено» / «Доставлено» / «Прочитано».
- **Доставлено / прочитано (чат отдела):** `read_meta.others_min_last_delivered_message_id` и **`others_min_last_read_message_id`** (минимум по остальным участникам); для своих сообщений «Доставлено» / «Прочитано», когда все остальные отметили соответствующий id.
- **Упоминания:** JSON + **`team_message_mentions`**; кнопки «Упомянуть»; подсветка `@Имя`; **Notification API** при @ и фоновой вкладке (после разрешения в сайдбаре).
- **Закреп чатов:** в pivot `team_conversation_user.pinned_at`; `POST .../api/{conversation}/pin`.
- **Закреп сообщения в чате отдела:** `team_conversations.pinned_team_message_id`; менеджер/админ; баннер, кнопка 📌, Echo `team.room-pin`.
- **Ответы (один уровень):** `parent_team_message_id` + `reply_to` в API/Echo/UI; цитата выделенного фрагмента при ответе.
- **Вложения и превью ссылок:** `team_message_attachments`, `team_messages.link_preview`, `LinkPreviewService`, UI (скрепка, превью, картинки).
- **Реакции:** `team_message_reactions`, `POST .../messages/{id}/react`, быстрые эмодзи + `EmojiPicker`, Echo `team.message.reactions`.
- **Интеграции:** кнопка «📅 Встреча» → календарь с `?create=1&title=…`; в чате отдела «✓» → задача (`organization.posts.store`) с цитатой и ссылкой на беседу.
- Доступ к разделу организации (включая чат) завязан на модуль **`module_tasks`**.

## 3. Дополнительные идеи (расширение)

- **Черновики и отложенная отправка** — не терять текст при смене беседы. ✅ черновики в `sessionStorage`
- **Политика хранения** переписки (ретенция) + юридическое согласие.
- **Боты / webhooks** (уведомления в чат отдела из CI, CRM, мониторинга).
- **Метрики** (без чтения текста): активность по отделам, время ответа SLA.
- **Доступность**: клавиатурная навигация, screen reader.
- **PWA / офлайн-кэш** последних сообщений (осторожно с чувствительными данными).
- **RTL / мультиязычность** интерфейса.

## 5. Поэтапная реализация

### Фаза 1 — быстрый выигрыш без тяжёлой схемы БД

| Приоритет | Что делаем | Заметки | Статус |
|-------------|------------|---------|--------|
| Высокий | **Черновики** при переключении бесед | `sessionStorage` по ключу `draft:team_conv:{id}`. | ✅ |
| Высокий | **`client_message_id`** | UUID на клиенте, уникальный индекс в БД. | ✅ |
| Средний | **Фильтры списка бесед** | UI + query-параметры. | ✅ |
| Средний | **Mobile-first** | Touch targets, нижняя панель ввода. | ✅ |

| Высокий | **@упоминания** | JSON + таблица + UI + браузерные уведомления. | ✅ |

## 7. Риски и нюансы (напоминание)

- **Пользователь без `company_id`**: ЛС и контакт-лист по компании недоступны; поведение зафиксировать в продукте.
- **Смена `company_id` у пользователя**: обновить или очистить участие в чатах старой компании.
- **Модуль организации**: сейчас чат завязан на `module_tasks`; при необходимости выделить отдельный флаг `module_team_chat`.
- **Офлайн / PWA**: баланс удобства и утечки данных на устройстве.

## 8. Долгосрочный бэклог (вне фаз 1–4)

Пункты, сознательно **не входящие** в текущий scope фаз 1–4; приоритизировать отдельно:

- Серверные **e-mail / mobile push** при офлайне (сейчас только браузерные @-уведомления).
- **Аудит** удаления сообщений / участников.
- Роль **«владелец чата отдела»** (отдельно от administrator/manager).
- **Голосовые** сообщения и **стикеры** (транскодирование, хранилище).
- **Вложенные треды** (сущность `team_thread`, счётчики, ответ на ответ).
- **Очередь** на тяжёлые вложения, **DLP**, **watermark**.
- **Экспорт HR** с журналом запросов.
- **Meilisearch / FTS** вместо `LIKE` при росте объёма.
- **Комнаты по проекту** (кросс-функциональные команды).
- Ретенция, боты/webhooks, метрики SLA, PWA-офлайн, RTL.
