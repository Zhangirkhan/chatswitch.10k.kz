 

## 1. Пункт чеклиста

**Вопрос:** системное сообщение в чате (`direction=system`) записывается в БД, отображается в UI и не уходит в WhatsApp?

**Вывод.** Всё в порядке: автотесты `tests/Feature/Chat/SystemMessageTest.php` (PHPUnit, SQLite в памяти) показывают, что при `ChatService::logSystemMessage` создаётся запись с `direction=system`, поля `whatsapp_session_id` и `whatsapp_message_id` пустые, текст сохраняется; для такого сценария `SendOutboundMessageJob` в шину не ставится; при прямом вызове `SendOutboundMessageJob::handle` для записи `direction=system` исходящих HTTP к WhatsApp нет, что согласуется с кодом джобы (отправка только для `outbound`).

Для воспроизведения прогона из корня репозитория выполняется команда:

```bash
./vendor/bin/phpunit tests/Feature/Chat/SystemMessageTest.php
```

Ожидаемый результат прогона: успешное завершение трёх тестов (восемь утверждений). Исходный код тестов — `tests/Feature/Chat/SystemMessageTest.php`. Реализация — `app/Services/ChatService.php`, метод `logSystemMessage`. Ограничение отправки в WhatsApp — `app/Jobs/SendOutboundMessageJob.php`. Отображение системного типа в UI — `resources/js/Pages/Chats/Partials/ChatMessage.vue` (`direction === 'system'`).

**Примечание.** Рендер в браузере и отсутствие текста системного сообщения в WhatsApp на стороне клиента на рабочей стенде в этом прогоне не проверялись; для полного аудита имеет смысл визуально проверить ленту после действий вроде смены отделов или назначений и тестовый контакт в WhatsApp.

**Приложить скриншоты.** Лента чата ChatSwitch с видимой системной строкой после смены отделов или назначений. Отдельно скриншот переписки WhatsApp на стороне клиента, подтверждающий отсутствие текста системного сообщения.
